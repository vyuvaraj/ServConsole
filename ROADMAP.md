# ServConsole Product Roadmap

This document outlines the design phases, completed modules, and upcoming features for the **ServConsole** platform.

---

## Completed Foundations

### Phase 1: Unified Console Portal
* [x] **Lightweight Aggregation Engine**: Dynamic CORS proxy and status aggregator for all three Servverse pillars.
* [x] **Ecosystem Dashboard**: Glassmorphic, dark-theme layout displaying overall latency and online status.
* [x] **Gateway Editor**: Live route list display and declarative edit sync to `ServGate/config.json`.
* [x] **WASM Middleware Hot-Swap UI**: Instant uploading of compiled WebAssembly modules directly into ServGate memory.
* [x] **Consistent Hash Ring Visualization**: Canvas circular layout showing node ring placement and key route tracer query.
* [x] **Distributed Observability Trace Flow**: Parse and visualize nested OpenTelemetry trace timelines.

---

## Planned Enhancements

### Phase 2: SQL & DB Schema Inspector (Q3 2026)
* [x] **DB Schema ORM Viewer**: Visual diagram of compiled database schemas generated by `serv-lang` compiled actors.
* [x] **SQL Query Workbench**: Light, secure web console to run queries against connected database adapters (Postgres, SQLite, MySQL).
* [x] **Migration Auditing**: Track schema revisions and delta logs directly from the UI.

### Phase 3: Cluster Operations & Repair Panel (Q4 2026)
* [x] **Erasure Coding Health**: Visual indicator of data shards vs. parity shards in ServStore cluster.
* [x] **Rebalance trigger**: Interactive action button to initiate P2P data healing and hash ring rebalancing.
* [x] **Replication Lag metrics**: Monitor real-time delta lags for cross-region replication configurations.

### Phase 4: Enterprise Access Control & Audit Logs (Q1 2027)
* [x] **Console SSO**: Integrated OIDC/OAuth2 and LDAP user sign-ins.
* [x] **User Policies (RBAC) Editor**: Create and apply granular user security policies for S3 buckets and STOMP topics.
* [x] **Audit Logs Dashboard**: Track administrative operations (e.g. WASM hot-swaps, route creations) in a secure immutable log table.

### Phase 5: Ecosystem Integration Depth (Q2 2027)
* [x] **Service Discovery Config**: Replace hardcoded localhost ports with a `services.json` or `SERVVERSE_DISCOVERY` env-var discovery map â€” prerequisite for Docker Compose and Kubernetes deployments.
* [x] **Shared OTel Collector Config**: Surface a single `SERV_OTLP_ENDPOINT` setting propagated to all connected services; enables unified cross-service trace correlation in one UI.
* [x] **ServQueue Topic Admin**: Full WAL inspection, delayed message queue view, and WASM transform debug panels in the dashboard.
* [x] **ServGate Multi-Replica Config Sync**: Write route configurations to a ServStore-backed distributed bucket (`serv-config`) instead of local JSON file.
* [x] **Cross-Service Dependency Graph**: Visual interactive map showing which Serv services depend on which infrastructure components (queues, buckets, routes).
